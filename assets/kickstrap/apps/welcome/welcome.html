How to use apps

How to Use Apps

Let's try loading an app already packaged with Kickstrap. Open your settings.less file and uncomment the app "updater" and pinesnotify (remove the "//"s).

Updater will talk to the Kickstrap servers and determine if your version is up-to-date. Pines Notify is a messaging system which Updater and other apps use to display messages.

Now try dragging a new app into your apps folder. In the Kickstrap folder, we've given you qunit.zip as an example. Drag this to your apps folder and unzip its contents there. Now, add this to the comma-separated list in settings.less. When you restart the page, you should see the qUnit unit tester pop up like this window.

Tip: if you turned the caching on in settings.less, you'll need to type "clearCache()" in the console to refresh the page.

How to Change Themes

You can also customize Kickstrap with themes made just for Bootstrap. Kickstrap includes a handful of these standard thanks to Thomas Park of Bootswatch.

Changing a theme is much like changing apps. Simply open settings.less and make sure the @import line for your chosen theme is uncommented (has no slashes in front of it.)

Now, just save and view your site. Remember, if you have caching on, you'll need to open the console and type "clearCache()"

Using the Console

The console is like an instant messenger between you and Kickstrap. We recommend having it open while you develop. Google Chrome has a clean, simple console but some prefer developing in Firefox with Firebug or its native inspector.

In Chrome, you can open the console by pressing command + shift + j on a Mac or ctrl + shift + j on Windows.

Here, Kickstrap will tell you if you're caching your styles, what apps you are running, and any messages written with the consoleLog() function.

Click at the very bottom of the console to type into it. For example, try typing kickstrap.hello.

There are many reasons you'd want to tell Kickstrap to keep quiet at times. You can do this by commenting out the @import line for the console in settings.less.

Setting your Directory

By default, Kickstrap works from the root directory. For example, if your site is "mysite.com," your root directory is "http://mysite.com/" or just "/".

However, you may want to host Kickstrap at a subdirectory like "http://mysite.com/blog/" or just "/blog/". In this case, change @rootDir in settings.less to "/blog/". Keep in mind, this will not affect resources linked directly in your index.html file like less.js, so remember to either add "/blog/" to those <script> and <link> tags as well, or call scripts and stylesheets via javascript to use the rootDir variable. (See this file in /Kickstrap/apps/welcome/ for an example.)

Using the Console Designing your Site

Designing your Site

The only main file we haven't mentioned is theme.less. This will automatically override any style on your site, even your chosen theme. This is intended to make it easy to start with a look and make it your own. If you don't want to use a prepackaged theme, just comment out the @import line in settings and you'll have a standard installation of Bootstrap.

If you're not familiar with writing in LESS, just write in CSS, that works too. However, LESS is much more efficient and is easy to learn.

As you can see, we've designed Kickstrap to make you more efficient and focused. That's all for now. To learn more, visit GetKickstrap.com.
